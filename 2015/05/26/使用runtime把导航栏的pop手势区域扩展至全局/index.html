使用runtime把导航栏的pop手势区域扩展至全局

随着手机屏幕越来越大，软件左上角的返回按钮越来越难点到，导航栏自带的pop手势也必须从屏幕左边缘触发，现在介绍一种最简单的扩展pop手势区域的方法。你只需要继承
UINavigationController，然后在viewdidload里加上下面这段代码

UIGestureRecognizer *gesture = self.interactivePopGestureRecognizer;  
   gesture.enabled = NO;  
   UIView *gestureView = gesture.view;  
   UIPanGestureRecognizer *popRecognizer = [[UIPanGestureRecognizer alloc] init];  
   popRecognizer.delegate = weakSelf;  
   popRecognizer.maximumNumberOfTouches = 1;  
   [gestureView addGestureRecognizer:popRecognizer];  


   NSMutableArray *_targets = [gesture valueForKey:@"_targets"];  
   id gestureRecognizerTarget = [_targets firstObject];  
   id navigationInteractiveTransition = [gestureRecognizerTarget valueForKey:@"_target"];  
   SEL handleTransition = NSSelectorFromString(@"handleNavigationTransition:");  
   [popRecognizer addTarget:navigationInteractiveTransition action:handleTransition];

现在就可以实现手势了，但是如果当前页面为根视图还是会有bug，再加上下面的代码判断导航栏的栈里只有一个视图则不触发方法

- (BOOL)gestureRecognizerShouldBegin:(UIGestureRecognizer *)gestureRecognizer  
{  

return self.viewControllers.count != 1 && ![[self valueForKey:@"_isTransitioning"] boolValue];  
}
